# Дизайн-спека: лендинг эксперта Сымбат

**Дата:** 2026-06-14
**Проект:** `karinagroup2023-oss/symbat-website`
**Статус:** утверждён пользователем (этап brainstorming завершён)

## 1. Суть и позиционирование

Сымбат — бухгалтер-аналитик и налоговый консультант для МСБ Казахстана. Ключевое
отличие от конкурентов: не массовое «ведение бухгалтерии по конвейеру», а **бутиковый
эксперт по диагностике учёта и снижению налоговых рисков**.

**Концепция лендинга (B + C):** в ядре оффера — фирменный продукт «Паспорт безопасного
перехода ИП → ТОО» (ни один из КЗ-конкурентов так не упакован), поданный через личный
премиальный тон и фото Сымбат для доверия. Ведущая эмоция — **уверенный безопасный рост**,
а не страх; боль клиента используется ниже по странице, но в герое — позитивная рамка.

**Конкуренты (для отстройки):** mybuh.kz, payda.me, finance-service.kz, gbconsult.kz,
asistent.kz — массовые онлайн-сервисы, обезличенные, «дёшево сдадим отчёт». Мы —
противоположность: персональный эксперт, премиум, снижение рисков.

## 2. Цели и метрики

- **Главное целевое действие:** написать в **WhatsApp** (кнопка с предзаполненным сообщением).
- **Доп. каналы:** Instagram, Telegram.
- Форма-заявка и звонок в v1 **не используются** (по выбору пользователя).
- Успех = посетитель доходит до блока контактов и жмёт WhatsApp.

## 3. Объём (scope)

- **Одностраничный лендинг** (single-page, якорная навигация).
- **Двуязычность:** русский + казахский, переключатель RU/KZ. RU — основной, KZ —
  полный перевод. Тексты хранятся в словаре локалей (см. §7).
- Без блога, личного кабинета, CMS, оплаты. YAGNI.

## 4. Стек и деплой

- React 19 + TypeScript + Vite + Tailwind CSS v4 (как в существующих проектах Karina Media).
- Деплой: Vercel.
- Шрифты: Google Fonts — **Playfair Display** (заголовки, сериф) + **Inter** (текст).
  Оба с поддержкой кириллицы и казахских глифов (ә, ғ, қ, ң, ө, ұ, ү, і).

## 5. Визуальный стиль (утверждён)

**Тёмный изумруд + золото**, премиальная база.

| Токен | Значение | Применение |
|---|---|---|
| `--emerald-900` | `#0b2c22` | глубокий фон/градиент |
| `--emerald-800` | `#0f3d2e` | основной фон секций |
| `--emerald-700` | `#185640` | градиент/карточки |
| `--gold` | `#c8a04a` | акценты, рамки, лейблы |
| `--gold-light` | `#e6c473` | акцент в заголовках |
| `--cream` | `#f7f3ea` | основной текст на тёмном |
| `--whatsapp` | `#27c08a` | главная CTA-кнопка |
| `--muted` | `#9fc3b4` | вторичный текст |

- Заголовки — Playfair Display 700/800. Текст — Inter 400–600.
- Скруглённые карточки, тонкие золотые рамки (`#c8a04a` ~25–40% прозрачности), мягкие тени.
- Hero — радиальный градиент изумруда. Достаточно воздуха, премиальная сдержанность.
- Доступность: контраст cream-на-emerald и emerald-на-gold проверить ≥ AA.

## 6. Структура страницы (утверждена, сверху вниз)

1. **Header** — лого «Сымбат», якорная навигация, переключатель RU/KZ, кнопка WhatsApp. Sticky.
2. **Hero** — заголовок «Безопасный переход ИП → ТОО», подзаголовок, фото Сымбат,
   бейдж флагмана, кнопка WhatsApp + ссылка на «Паспорт перехода», 3 микро-факта доверия.
3. **«Знакомо?»** — 7 проблем клиента карточками (обороты выросли; есть бухгалтер, но нет
   уверенности; уведомление от налоговой; заблокировали счёт; хаос после смены бухгалтера;
   неясны налоги после перехода; риск доначислений).
4. **Почему обычный бухгалтер не спасает** — сравнение «обычный бухгалтер ↔ Сымбат».
5. **Услуги** — 5 карточек: переход ИП→ТОО · второй контур контроля · восстановление учёта ·
   решение проблем с налоговой · финотчётность для банков.
6. **⭐ Флагман «Паспорт безопасного перехода ИП → ТОО»** — что входит: налоговые расчёты и
   выбранный режим; карта рисков и исправлений; правила работы с банком и назначениями
   платежей; требования к первичке и ЭСФ; календарь отчётности. Главный оффер.
7. **Как я работаю** — процесс из 4 шагов: диагностика → сценарии расчёта → внедрение →
   контроль первых периодов.
8. **Для кого** — ниши (услуги · опт/розница · фармация) + профиль идеального клиента
   (собственник ИП/ТОО, оборот от 10 млн ₸/год). Явно: строительство и логистика — не профиль.
9. **Об эксперте** — фото, подход, цитата: «Я показываю собственнику реальную картину
   бизнеса, нахожу риски заранее и помогаю расти легально — без убытков, доначислений и хаоса».
10. **Финальный CTA + контакты** — заголовок-призыв, WhatsApp (главное), Instagram, Telegram.
11. **Footer** — имя, контакты, © год, мелкий дисклеймер (не публичная оферта).

## 7. Архитектура кода (компоненты)

Каждая секция — отдельный самодостаточный компонент в `src/sections/`, принимающий тексты
из текущей локали. Точечные границы, чтобы секции читались и менялись независимо.

```
src/
  main.tsx, App.tsx
  i18n/
    LocaleContext.tsx      # state RU|KZ, хук useT()
    ru.ts, kz.ts           # словари текстов по секциям
  sections/
    Header.tsx  Hero.tsx  Pains.tsx  WhyNotOrdinary.tsx
    Services.tsx  Passport.tsx  Process.tsx  Audience.tsx
    About.tsx  FinalCta.tsx  Footer.tsx
  components/
    WhatsAppButton.tsx     # формирует ссылку wa.me с предзаполненным текстом
    LangSwitch.tsx  SectionHeading.tsx  Card.tsx
  config/site.ts           # контакты, ссылки, флагман-факты (единый источник)
  styles/ (tailwind theme: цветовые токены §5)
```

**Поток данных:** локаль хранится в `LocaleContext`; `useT()` отдаёт строки из `ru.ts`/`kz.ts`.
Контакты и внешние ссылки — только в `config/site.ts`, компоненты их импортируют. Никакого
бэкенда: WhatsApp/Instagram/Telegram — внешние ссылки. Состояние минимально (локаль + мобильное меню).

## 8. Что нужно от пользователя до/во время вёрстки (контент)

- Полное имя/бренд для лого и блока «Об эксперте».
- Фото Сымбат (желательно вертикальное, хорошее качество).
- Реальные контакты: номер WhatsApp, ссылки Instagram и Telegram.
- Подтверждение/правка казахских переводов (термины налогов/учёта лучше выверить носителем).

До получения — собираем на аккуратных плейсхолдерах (фото-заглушка, демо-контакты), реальные
данные подставляем в `config/site.ts` и словари локалей. Кейсов/отзывов пока нет — доверие
строим через флагман-продукт, процесс, сравнение и цифры, без выдуманных отзывов.

## 9. Нефункциональные требования

- Адаптив: mobile-first, аккуратно на 360px и десктопе.
- Производительность: статический билд, ленивые шрифты, оптимизированное фото.
- SEO-минимум: корректные мета-теги (RU/KZ), Open Graph, заголовочная иерархия, lang-атрибут.
- Без аналитики/кук в v1 (можно добавить позже).

## 10. Явно вне объёма (YAGNI)

Блог, CMS, форма-заявка с бэкендом, онлайн-оплата, личный кабинет, калькулятор налогов,
интеграции с CRM. Не делаем в v1.
